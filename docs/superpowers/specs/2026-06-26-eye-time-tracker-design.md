# Windows 用眼时间统计小程序设计

## 目标

做一个 Windows PC 端本地小程序，用来统计每天的电脑用眼时间。它不需要登录、不联网、不做账号体系。程序默认随 Windows 开机启动，平时常驻系统托盘，需要查看时打开一个简单窗口，显示今天、本周、本月累计用眼时间。

## 核心需求

用户想知道每天实际看电脑屏幕的总时长。这个时间不仅包括键盘、鼠标操作下的工作状态，也包括播放视频、听课、看回放等可能长时间没有键鼠动作的场景。

当天累计达到 5 小时 30 分钟时，程序弹出一次提醒。

## 产品形态

- Windows 本地桌面程序，优先使用 .NET 实现。
- 默认以系统托盘程序运行。
- 托盘图标可以打开主窗口，也可以退出程序。
- 默认开机自启，避免漏记当天用眼时间。
- 无登录、无账号、无云同步。

## 计时模型

计时必须使用“短片段累计”，不能因为某次检查发现用户处于活动状态，就把之前一大段时间都算作用眼时间。

程序保存上一次计时检查的时间。每次检查时：

1. 读取当前状态。
2. 计算距离上一次检查过去了多少秒。
3. 判断刚过去的这一小段时间是否应该计入。
4. 如果应该计入，只累加这一小段的实际秒数。
5. 更新上一次检查时间。

默认检查间隔建议为 10 秒左右。如果测试确认没有任何可感知负担，可以缩短到 5 秒；但实现上必须避免让系统出现周期性卡顿。

例子：

- 如果第 4 分钟有一次键鼠动作，那么它最多让第 4 分钟之后约 3 分钟内的短片段计入。
- 如果下一次动作发生在第 58 分钟，中间第 7 分钟到第 58 分钟之间没有音频输出，也没有新的键鼠动作，就不能计入。
- 程序不能把整个小时回填成用眼时间。

## 用眼时间规则

一个短片段要计入用眼时间，必须先满足这些基础条件：

- 程序正在运行。
- Windows 会话未锁定。
- 系统没有睡眠或挂起。

同时满足以下活动条件之一：

- 最近一次键盘或鼠标输入距离当前时间不超过配置的空闲阈值。
- 系统当前有音频输出，并且音频计时开关已开启。

默认设置：

- 键鼠空闲阈值：3 分钟。
- 音频计时：开启。
- 每日提醒阈值：5 小时 30 分钟。
- 重复提醒：关闭，每天只提醒一次。
- 开机自启：开启。

## 性能约束

程序应该尽量没有存在感，不能因为后台统计造成明显系统负担。

- 优先使用低成本 Windows API 判断空闲时间，避免使用全局键盘/鼠标钩子。
- 计时检查放在后台线程或后台定时器中，不阻塞界面线程。
- 每次检查只做很少的计算。
- 不每次检查都写磁盘；先在内存中累加，再定期保存，例如每分钟、日期切换时、程序退出时保存。
- 主窗口隐藏时不频繁刷新界面；窗口打开时也只做温和刷新，例如 5 到 10 秒一次。
- 音频输出检测也要保持轻量，必要时可以比键鼠空闲检测更低频。

目标是用户感受不到卡顿、冻结、周期性停顿或明显 CPU 峰值。

## 统计功能

程序按本地日期保存每天累计秒数。

主窗口显示：

- 今天累计用眼时间。
- 当前是否正在计时。
- 本周累计用眼时间。
- 本月累计用眼时间。

本周统计按本地日历计算，默认周一作为一周开始。

## 提醒规则

当今天累计时间达到 5 小时 30 分钟：

- 弹出一次 Windows 通知。
- 将当天标记为已提醒。
- 当天不再重复提醒。
- 第二天自动重置提醒状态。

如果程序关闭期间已经超过阈值，再次打开后可以补发当天唯一一次提醒。

## 数据存储

第一版使用本地轻量 JSON 文件保存数据，因为数据结构简单，也方便检查和迁移。

需要保存：

- 每个日期的累计秒数。
- 每个日期是否已经提醒。
- 设置项：键鼠空闲阈值、是否启用音频计时、提醒阈值、是否开机自启。

如果数据文件缺失或损坏，程序应使用默认设置启动，并尽量保留仍然有效的历史记录，不能直接崩溃。

## 主窗口

主窗口保持简单实用：

- 明确显示今天累计时间。
- 显示当前状态，例如“正在计时”或“暂停计时”。
- 显示本周、本月合计。
- 提供设置：开机自启、键鼠空闲阈值、音频计时开关、提醒阈值。

它不是宣传页，也不需要复杂视觉效果。打开要快，运行要安静。

## 技术方案

推荐技术栈：

- .NET Windows 桌面程序。
- WinForms 或 WPF 实现托盘和主窗口。
- 使用 Windows 原生空闲时间 API 判断键鼠空闲状态。
- 使用 Windows 音频会话或音频端点能力判断是否有音频输出。
- 数据保存在当前用户的应用数据目录。
- 通过当前用户的 Windows 启动项实现开机自启。

实现模块应拆分为：

- 活动状态检测。
- 短片段计时累计。
- 本地数据保存。
- 提醒判断和通知。
- 托盘与窗口界面。

这样计时逻辑可以脱离界面单独测试。

## 测试要求

需要覆盖：

- 短片段累计不会回填长时间空闲。
- 键鼠动作只影响之后空闲阈值内的片段。
- 音频输出可以在没有键鼠动作时继续计时。
- 本地日期切换后当天统计正确滚动。
- 本周、本月统计能从每日记录正确汇总。
- 每日提醒达到阈值后只出现一次。
- 数据文件缺失或损坏时程序不崩溃。

手动验证需要包括：

- 程序隐藏到托盘后仍能继续统计。
- 打开和关闭主窗口不影响后台统计。
- CPU 占用保持很低。
- 采样过程不会造成肉眼可感知的系统卡顿。
